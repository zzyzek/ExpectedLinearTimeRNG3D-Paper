\begin{abstract}
  An expected linear time algorithm for computing the relative neighborhood graph (RNG) for points in Euclidean space,
  distributed uniformly in a unit three dimensional cube, is given.
  For each point, potential neighbors for an RNG connection are limited to an expected small bounding volume.
  Volumes relative to the considered point are bounded through half plane partitions,
  constructed from the point and its neighbors,
  that remove regions where no RNG edge connections to the point can occur.
  %The bounding volume is found by fencing in enclosing a convex hull 
  %where the bounding volume is found through a plane partition heuristic.
  The small bounding volume provides an expected constant number of neighbors to consider, making
  each single point RNG calculation $O(1)$, providing the $O(n)$ expected runtime when enumerating through all points.
  %The $K$ surrounding potential neighbors are then tested with
  %a naive $O(K^3)$ algorithm.
  %The RNG has finite degree for random point distributions in three dimensions,
  %resulting in an expected finite $K$, making $O(K^3) = O(1)$, providing 
  %the linear expected time algorithm when enumerating through all points.
  To the author's knowledge, this improves on the previously best known $O(n \log n)$ in three dimensions.
\end{abstract}

\section{Introduction}

Let $P = (p _ 0, p _ 1, \dots, p _ {n-1})$ be a set of points in $\mathbb{R^3}$.
The relative neighborhood graph, $\text{RNG}(P)$, is an undirected, simple graph that connects two vertices,
$p _ i, p _ j$ if and only if there is no other vertex within the \emph{lune} between
them.

$$
\begin{array}{l}
  (p _ i, p _ j) \in \text{RNG}(P) \\
  \iff |p _ i - p _ j| \le \max _ { k \ne i,j } ( |p _ i - p _ k|, |p _ j - p _ k| )
\end{array}
$$

Figure \ref{TK} shows a highlighted 2D example of a connections and precluded connection.

Call the relative neighborhood graph around a point to be $\text{RNG}(P,p)$.
That is, the subset of the relative neighborhood graph that only has an edge that includes $p$.

When the points are drawn from a uniform random process in the unit cube, the maximum
degree of the relative neighborhood graph is bounded by a constant.

\section{Motivation and Structure}


For $P = (p _ 0, p _ 1, \dots, p _ {n-1})$,
a rough overview of the algorithm proceeds as follows:

\begin{itemize}
  \item Create a grid, where each cell size of $n^{-1/3}$
  \item Bin points into grid cells, allowing for multiple points in a bin
  \item For each point, $p$:
    \begin{itemize}
      \item keep adding nearby points from adjacent grid cells and discard
        further away points whose connection to $p$ would be precluded by
        the existence of the nearby point
      \item keep extending the radius of a bounding cube, at grid cell increments,
        until the collected nearby points have precluded connections from
        all other points outside bounding cube to $p$
      \item once a bounding cube has been found that has precluded a connection
        to $p$ from all other points outside, add nearby points outside
        the bounding cube that might preclude a connection of inside points to $p$
      \item run a naive relative neighborhood graph, \\
        $\text{RNG}(P,p)$, on all collected nearby points
        to determine connections to $p$
    \end{itemize}
\end{itemize}

A coarse grained plane partition heuristic is used to exclude further away points from consideration.

Given points $p$ and $q$,  any point, $w$, that lies on the other side
of the plane with normal $\frac{(q-p)}{|q - p|}$ that passes through $q$, cannot connect to $p$, as $q$ would
be in any lune that $p$ and $w$ would create.
Figure \ref{TK} gives a graphical representation of points excluded by this plane heuristic
in two dimensions.

The plane partition heuristic is coarse, as the entire
region that $q$ excludes from $p$ is more complicated (see Figure \ref{TK}).
While the plane partition heuristic is coarse, it's easy to caclulate and provides a good enough
way to exclude large portions of points so that it can be used
as the foundation for an expected linear time algorithm.

%The plane partition heuristic excludes points, but
To maintain a linear time algorithm, we need to
avoid processing points individually and
to implicitely exclude large collection of points that can never connect to the our anchor point $p$.
To do this, we consider a current grid cell collection of integral radius $R \in \mathbb{Z}$, centered
on the grid cell where anchor point $p$ resides.
If the tangent planes that passes through neighbor $q$, and in direction $(q-p)$ within the grid cell collection,
partition each of the six sides of our cube of radius $R$, we can be assured that any points outside
of the grid cell collection will never be able to connect to $p$.

A grid cell collection of a radius $R$ is called a \emph{fence} of radius $R \in \mathbb{Z}$.
When $R$ is understood from context, this will be called just a \emph{fence}.
If the sides of the \emph{fence} has been partitioned, with anchor point $p$ and neighbors $q$ within
the \emph{fence}, the \emph{fence} will be called \emph{secured}.

Points outside of the \emph{fence} will never be able to connect to anchor point $p$ but they still
might be in the lune of some points within the \emph{fence}.
For this reason, when collecting points for the naive relative neighborhood graph calculation, some
points outside of the fence will need to be added, as they can sabotage potential edges within
the \emph{fence}.

This will be done by considering a larger radius of the current \emph{fence} to add points
but the increase in $R$ will shown to be bounded by a constant factor, allowing overall linear time to be maintained.

Before discussing the algorithm in detail, preliminary structural results will be discussed.

If $w$ is on the other side of the plane that passes through $q$ and with tangent in the direction of $(q-p)$,
then $q$ will always be in the lune created by $p$ and $w$ (see Figure \ref{TK}):


\begin{lemma}[Plane Partition Heuristic]
  \label{lemma:plane}
  if $p,q,w \in \mathbb{R}^3$ with:
  %w \in H = \{ u \in \mathbb{R}^3 | (q-p) \cdot (u - q) \ge 0 \}
  $$
  \begin{array}{l}
  w \in H ^ {+} _ {p,q} = \{ u \ :  \ (q-p) \cdot (u - q) \ge 0 \}
  \end{array}
  $$
  then
  $$
  \begin{array}{l}
    q \in \text{Lune}(p,w)  = \\
      \ \ \ \{ v \ : \ |v-p| \le |p-w| \ \& \ |v-w| \le |p-w| \}\\
  \end{array}
  $$
\end{lemma}

\begin{pf}
  TK
\end{pf}

The plane partition heuristic allows us to construct a convex hull from neighboring
points that will bound the area that needs to be tested for relative neighborhood graph
neighbors of $p$.

Only points within the convex hull have the potential to connect to $p$:
%need to be considered for a relative neighborhood graph connection to $p$:

\begin{lemma}[Convex Hull Heuristic]
  \label{lemma:hull}
  Let $p \in \mathbb{R}^3$, $Q = (q _ 0, q _ 1, \dots, q _ {m-1}) \subset P$ and $\mathbf{C}$ be the
  compact convex hull created from $H _ i ^ {-} = \{ u \in \mathbb{R}^3 | (q _ i - p ) \cdot (u - q _ i) < 0 \}$,
  then $(w,p) \in \text{RNG}(P,p) \to w \in \mathbf{C}$
\end{lemma}

\begin{pf}
  By Lemma \ref{lemma:plane}, all $w$ outside of $\mathbf{C}$ will have some $q _ i \in Q$ in the $\text{Lune}(w,p)$, so cannot have an RNG edge to $p$.
\end{pf}

Lemma \ref{lemma:hull} only guarantees that RNG connections to $p$ fall within the hull, $\mathbf{C}$.
Not every point within $\mathbf{C}$ might connect to $p$ and points outside of $\mathbf{C}$ might still influence
connections to $p$.
For example, Figure \ref{TK} shows a potential connection between two points within the convex
hull but is thwarted by a point outside the convex hull that is within the lune created by the pair.

Points outside of the convex hall might influence connections to $p$ but we can bound how
far these points can be.
The maximum distance to $p$ and the convex hull bounds what points can influence connections to $p$ to
that distance:

\begin{lemma}[Convex Hull Extension Heuristic]
  \label{lemma:extension}
  Let $p, \mathbf{C}$ be the anchor point and convex hull as detailed above.
  Let $R _ {\text{max}} = \max _ {u \in \mathbf{C}} |u-p|$ be the maximum distance of the convex hull boundary to $p$.
  All points that could affect $\text{RNG}(P,p)$ lie within a sphere of radius $R _ {\text{max}}$, centered at $p$.
\end{lemma}

\begin{pf}
  The maximum lune radius for any neighbor to $p$ is bounded above by $R _ {\text{max}}$.
  Any point, $w$ such that $|w-p| > D$, will be further away from the maximum lune radius
  and so can't influence any connection within $\mathbf{C}$.
\end{pf}

The above already suggests a linear time algorithm by starting at a grid cell that $p$ resides in
and adding neighboring points from neighboring grid cells until a compact convex hull has been achieved
that meets some maximum radius criteria.
All points within a sphere centered at $p$ can then be added and a naive relative neighborhood graph algorithm
can be run on the reduced neighbor set.
It can be shown that for a constant number of neighbors, the maximum distance of $p$ to the convex hull is bounded,
giving an expected linear time algorithm for random points in the unit cube.

While there is an expected constant number of neighbors that make up the convex hull, the convex hull's representation
above needs to be converted to points via a vertex enumeration procedure, inflating the runtime constants were a
naive vertex enumeration algorithm to be used.
There may be ways of speeding the vertex enumeration up further by considering the structure of the problem in question
but this methodology is not pursued further in this paper.

Instead, a bounding box cube is used as a proxy for the convex hull.
The bounding box cube, called a \emph{fence}, has certain key point points on each face that will be used
to indicate when a cutting plane has partitions portions of the fence from the anchor point, $p$.
Once all portions have been partitioned, a process that will be called \emph{securing the fence},
there will be an embedded convex hull within the fence and only the points within the fence need to be
considered for connections.

For the anchor point $p$, only $RNG(P,p)$ connections can happen within the secured fence but, as alluded
to with \ref{lemma:extension}, other points outside of the secured fence might influence connections within.
We will see that a constant factor of fence radius need be considered to add additional points to the naive
relative neighborhood graph computation.

A secured fence necessarily implies a compact convex hull embedded within the fence but the other direction
is not true in general, as there can be a convex hull whose volume is completely contained within the fence but whose
partition planes do not secure the fence.
We will see that this has no effect on the expected linear runtime as securing the fence for an individual
anchor point is expected to use a constant number of neighbors.

\begin{algorithm}
  \caption{Relative Neighborhood Graph for single point, Naive method}\label{alg:RNGNaive}
  \begin{algorithmic}[1]
    \Procedure{RelativeNeighborhoodGraphPointNaive}{L, p}
      \For{ $q \in L / \{p\}$ }
        \State $\text{isConnected } = \text{ \emph{False}}$
        \For{ $u \in L / \{p,q\}$ }
          \If{ $\text{InLune}(p,q,u)$ }
            \State $\text{isConnected } = \text{ \emph{False}}$
          \EndIf
        \EndFor
        \If{ $\text{isConnected } = \text{ \emph{True}}$ }
          $E = E \cup (p,q)$
        \EndIf
      \EndFor

      \State $\text{return } E$
    \EndProcedure
  \end{algorithmic}
\end{algorithm}


\begin{algorithm}
  \caption{Relative Neighborhood Graph, Secure Posts on Increasing Fence (SPoIF)}\label{alg:RNGSPoIF}
  \begin{algorithmic}[1]
    \Procedure{SPoIF\_RNG}{P}
    \State \vspace{0.5em} $E = \{\}$
    \State \vspace{0.5em} $N = |P|, \ \ \ \ N_s = \lceil N^{\frac{1}{3}} \rceil, \ \ \ \ s = \frac{1}{N_s}$,
    \State \vspace{0.5em} $B[0:N_s][0:N_s][0:N_s] = \emptyset$
    \For{$p \in P$}
      \vspace{0.5em} \State $\text{append}( B[ \lfloor N_s  p_x \rfloor ][ \lfloor N_s  p_y \rfloor ][ \lfloor N_s  p_z \rfloor ] , p )$
    \EndFor

    \For{$p \in P$}
      \vspace{0.5em} \State $E = E \cup \text{SPoIF\_RNGp}(B,P,p)$
    \EndFor

    \State $\text{return } E$

    \EndProcedure
  \end{algorithmic}
\end{algorithm}

\begin{algorithm}
  \caption{Single Point Relative Neighborhood Graph, Secure Posts on Increasing Fence, single point}\label{alg:RNGpSPoIF}
  \begin{algorithmic}[1]
    \Procedure{SPoIF\_RNGp}{B, P, p}

      \State Initialize $\text{FencePostCluster}$ to $\text{False}$, \\
        take $2 \times 2$ clusters of fence posts for each face

      \State $R _f = 0, R _ {\text{max}} = 0$

      \For{ $(R _ f \le N_s) \ \& (\text{\emph{False}} \in \text{FencePostCluster})$ }
        \For{ $q \in \text{GridPerimeterCells}(B, p, R_f) / \{p\}$ }
          \State $L = L \cup \{q\}$
          \State $R _ {\text{max}} = $ maximum of $R_{\text{max}}$ and grid cell distance that lune of $(p,q)$ intersects
        \EndFor

        \For{ $q \in \text{GridPerimeterCells}(p, R_f) / \{p\}$ }
          \State Define $\rho(u) = \text{sgn}((q-p) \cdot (q - u))$
          \For{ every cluster, $FPC$ }
            \If{ every $v \in FPC$, $\rho(v) = - \rho(p)$ }
              \State Mark cluster in $\text{FencePostCluster}$ as $\text{\emph{True}}$
            \EndIf
          \EndFor
        \EndFor
        $R _ f ++$
      \EndFor

      \For{ $R _ e = [R _ f+1 : R _ {\text{max}} ]$ }
        \For{ $q \in \text{GridPerimeterCells}(p, R_f) / \{p\}$ }
          \State $L = L \cup \{q\}$
        \EndFor
      \EndFor

      \State $\text{return } \text{RNGPointNaive}( P, B, p, L )$

    \EndProcedure
  \end{algorithmic}
\end{algorithm}

\section{Appendix.0}

For completeness, this section discusses the elementary but tedious calculations
for the plume shape and volume.
As a reminder, our goal is to show that increasing the grid fence by a fixed size
will almost surely encapsulate a compact convex hull made from an anchor point and its neighbors.
The convex hull guarantees that the relative neighborhood graph edges of the anchor point to
its neighbors need only consider candidate points within the convex hull (with some caveats
about extending the grid fence to include saboteurs, detailed in section \ref{TK}).

The logic chain is roughly:

\begin{itemize}
  \item For an anchor point, $p$, and a grid fence side, there will be a region that a neighbor
        point can fall in that will partition the fence side from the point, a process we call
        \emph{securing the fence}, where a secured fence necessarily implies a compact convex hull
        made from the points on the interior
  \item The volume that secures a fence side will turn out to be four sphere slices joined together,
        which we will call a \emph{plume}, where a sphere slice is the intersection of two perpendicular
        half planes and a sphere
  \item A lower bound estimate on the plume volume is a constant proportion of overall grid fence volume
  \item A non-trivial constant plume volume makes the question of how many neighbors to be expected
        to secure the overal six sided fence into a coupon collector-like problem, giving us an
        expected finite grid fence size
  \item Finally, approximatations on the four slices bundled together
        allow us to give coarse values for grid fence sizes
\end{itemize}

It should be stressed that the discussion and estimates are meant primarily to show existence of
a finite grid fence size.
We focus on securing a fence side by waiting for some single neighbor point to secure it.
%, which is equivalent to considering a four post fence side.
The Shrinking Posts on Increasing Fence (SPoIF) algorithm in this paper has a smaller expected grid fence
size before securitization
as points can secure parts of the fence through clusters of posts, increasing the admissible
region on each face side that can be secured.

The calculations done in this appendix are meant to show that we only need to consider a finite grid fence
size by providing correct, if horrible, bounds.
In this paper, we are satisfied with providing empherical results for grid sizes that SPoIF uses without
going into more rigorous analysis.



First consider a fixed point $q$ and a fixed but moveable point $\eta$.

We think of $\eta$ as a point on a plane with normal $\eta / |\eta|$
that intersects $q$ in the slice of $z=0$ to $z=1$.

As we vary $\eta$ we want to know what the shape is traced out.

The claim is that this is the surface of a sphere whose center is at $q/2$
of radius $q/2$.

To see this, consider $\eta = q/2 + (|q| / 2) v$, with $|v| = 1$.

Calculate dot product of the line from $q$ to $\eta$ with the line from the origin to $\eta$:

$$
\begin{array}{ll}
  & \eta = q/2 + (|q| / 2) v \\
  & (q/2 + (|q|/2) v) \cdot (q/2 + (|q|/2) v - q ) \\
  = & (q/2 + (|q|/2) v) \cdot ( -q/2 + (|q|/2) v ) \\
  = & -(1/4)|q|^2 + (|q|/2) ^2 |v|^2 \\
  = & 0
\end{array}
$$

Proving the claim.

We want to find the the minimum volume of the plume
for some point inside of a fence of radius $s$ (side length $2s$)
nested inside of a larger fence of radius $s(2k+1)$ (side length $2s(2k+1)$).

First consider a sphere of radius $R$ centered at $(-D,-D,-D)$ intersected with two
half planes, $H ^ + _ {x=0} = \{ u : u _ x \ge 0 \}$, $H ^ + _ {y=0} = \{ u : u _ y \ge 0 \}$.
For suitable $R$ and $D$, we want to estimate the volume.

If we take two tetrahedrons completely enclosed by the slice, the maximum diagonal of
the slice is $(R-\sqrt{2}D)$, with maximum straight side length (in the $x$-axis line and $y$-axis line) to
also be $(R-\sqrt{2}D)$ and slice half height to also be $(R-\sqrt{2}D)$.

The volume of the tetrahedron is then:

$$
(1/6) (R - \sqrt{2} D)^2 \sqrt{R^2 - 2 D^2}
$$

The volume of the two tetrahedrons stacked on top of each other:

$$
(1/3) (R - \sqrt{2} D)^2 \sqrt{R^2 - 2 D^2}
$$

More succinctly:

$$
\begin{array}{l}
  V _ {\text{slice}} > \frac{1}{3} (R - \sqrt{2} D)^2 \sqrt{R^2 - 2 D ^ 2}
\end{array}
$$

We will get concrete estimates below but as a quick confirmation, we can check to make sure that
this slice is a constant proportion of the fence it sits in.

We know that $R \propto (k s)$
and $D \propto (k s)$, for some $k \in \mathbb{Z}$ and grid cell size $s \in \mathbb{R}$.
Call $R = \alpha (k s)$ and $D = \beta (k s)$.

The plume will be four slices bundled together sitting inside of a $\gamma^3 (k s)^3$ volume.
Take $\alpha, \beta \in \mathbb{R}$ to be the smallest of the four slices:

$$
\begin{array}{ll}
  & \frac{ (4/3) (\alpha (k s) - \sqrt{2} \beta (k s))^2 \sqrt{ \alpha^2 (k s)^2 - 2 \beta^2 (k s)^2 } }{ \gamma^3 (k s)^3 } \\[10pt]
  = & \frac{ (4/3) (k s)^3 (\alpha - \sqrt{2} \beta )^2  \sqrt{ \alpha^2 - 2 \beta^2 } }{ \gamma^3 (k s)^3 } \\[10pt]
  = & \frac{ (4/3) (\alpha - \sqrt{2} \beta )^2  \sqrt{ \alpha^2 - 2 \beta^2 } }{ \gamma^3 } \\[10pt]
\end{array}
$$

as desired.

When choosing random points inside of the larger fence cube, we can think of the plume as the probability
a single point secures one side of the fence.
It is equivalent to talk about securing one side of the fence face with securing four posts on each of corners of
the fence face.
As a reminder, the Shrinking Posts on Increasing Fence (SPoIF) uses nine posts per face, equally distributed
along the face square.
Since there are more opportunities to secure clusters of posts on a fence face,
in addition to still retaining the features of the plume region, SPoIF will be able to secure the fence cube
more quickly.
We only consider the four post fence here to help simplify the calculation even though it inflates the expected
number of points we need to see to secure the grid fence.

Take the minimum plume value volume of the six directions and call it $V _ {\text{plume}}$.
The probability that a point lands in one of the plumes is bounded below by $V _ {\text{plume}} / V _ {\text{fence}}$.
Of the points that land in one of the plumes, the time we need to take to secure each face of the fence is now
a coupon collector-like problem with six coupons.

We can now plug in some real numbers to get coarse bounds on plume volumes, probabilities of landing in the critical region
and expected grid sizes.

As a reminder, we are considering a center grid cell, of side length $s$ where the anchor point, $p$,
resides, all of which is inside a larger grid fence of side length $2s(k+1)$, for some fixed $k \ge 0, k \in \mathbb{Z}$
as yet to be determined.

The smallest size a slice can be is if the anchor point is on the corner of the interior grid cell and to choose the grid fence corner
nearest to it.
For example, $p = (s/2,s/2,s/2)$, $q = ( s(k+1/2), s(k + 1/2), s(k+1/2) )$.

In such a case, the center point of the sphere is $(p + q)/2 = ( s(k+1)/2, s(k+1)/2, s(k+1)/2 )$,
with radIUs $R = |(p+q)/2 - p| = |(sk/2,sk/2,sk/2)| = (\sqrt{3}/2) sk$.
Here, $D = s(k+1)/2 - s/2 = sk / 2$ with $V _ {\text{fence}} = (s(2k+1))^3$.

This yields:

$$
\begin{array}{ll}
  \frac{ V_{\text{plume}} }{ V _ {\text{fence}}} > &  \frac{ \frac{4}{3} ( \frac{\sqrt{3}}{2} sk - \frac{\sqrt{2}}{2} sk)^2 ( \frac{3}{4}s^2 k^2 - \frac{1}{2} s^2 k^2 )^{\frac{1}{2}}  }{  s^3 (2k+1)^3  } \\[10pt]
  = & \frac{ s^3 k^3 ( \sqrt{3} - \sqrt{2} )^2 }{ 3 \sqrt{2} s^3 (2k+1)^3 } \\[10pt]
  = & \frac{ ( \sqrt{3} - \sqrt{2} )^2 }{ 3 \sqrt{2} (2+\frac{1}{k})^3 } \\[10pt]
  > & \frac{ ( \sqrt{3} - \sqrt{2} )^2 }{ 81 \sqrt{2}  } \ \ \ \ \ \ \ \ (k \ge 1) \\[10pt]
  > & 1 / 1198
\end{array}
$$

For points that land within one of the six plume region, we would expect this to be be a six coupon collector
problem with, at worst, equal probability.
The coupon collector problem grows roghly as $n \log(n)$  which, in our case of six sides of the fence to secure,
gives us an expected $11$ or more points before we secure the fence.

Each point has roughly a $6 / 1198$ chance of landing in one of the six plumes, so we would expect
roughly $11 / (6 / 1198) = 2196.3\dots$ number of points within the grid fence to appear before
securing the fence.

Each grid cell has roughly one point, so we would expect the number grid cells on a side to
be

$$
\begin{array}{ll}
  & (11 \cdot 1198 / 6)^{\frac{1}{3}} \\[10pt]
  = &  12.9\ldots \sim 13 \\[10pt]
\end{array}
$$

That is, we would expect $k \sim 6$, iterating roughly six to seven steps of increasing the grid fence size
until securing the fence in this way.
